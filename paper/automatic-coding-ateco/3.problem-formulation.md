# Problem Formulation

## Section Goal

Define the task precisely and early, so the rest of the paper reads as a response to a well-posed problem rather than as an exploratory benchmark.

## Core Definition

Input:

- a short free-text description of an economic activity

Output:

- a ranked list of candidate ATECO codes, ideally at the 5-digit level for comparison with CIRCE labels

## Key Framing Decisions

- The problem should be presented as ranking or retrieval, not only as classification.
- The operational object is a shortlist of plausible codes.
- Top-k retrieval is justified by the real coding setting, where users or staff choose among candidates.
- The problem should be tied to NSI constraints: usability, limited infrastructure, need for validation, and transition across taxonomy revisions.

## Suggested Subsections

### 3.1 Task definition

Map a free-text description to a ranked candidate set of ATECO codes.

### 3.2 Coding as a ranking problem

Explain why top-k retrieval is more faithful than top-1 classification to practical coding workflows.

### 3.3 Operational requirements in NSIs

State explicitly:

- shortlist usefulness
- deployability with moderate computational resources
- compatibility with expert review
- ability to adapt to taxonomy updates with lower maintenance burden

## What To Avoid

- Do not leave the top-k logic implicit until the results section.
- Do not present the contribution as a fully autonomous coding system if the paper mainly supports decision support.
