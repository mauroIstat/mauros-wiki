# Problem Formulation

The previous sections motivated automatic coding of economic activities as a semantic retrieval task in Official Statistics. We now make this formulation explicit. The problem considered here is not simply that of assigning a single code to a text and evaluating performance through standard top-1 metrics. In realistic coding settings, the practical objective is often to support the identification of plausible categories from short and heterogeneous textual descriptions, while preserving compatibility with the operational requirements of official statistical production. For this reason, the aim is to define a methodological framework for economic-activity coding that treats the task as ranking over the ATECO label space.

## 3.1 Task definition

Let a query be a short free-text description of an economic activity expressed in natural language. Let the target space be the set of admissible ATECO categories at the chosen level of analysis. In the present study, the main target level is the 5-digit ATECO category, which is also the level at which CIRCE labels are available for the real-world benchmark.

The coding task can then be defined as follows: given a textual query, produce an ordered set of candidate ATECO categories, ranked according to their estimated semantic compatibility with the query. Under this formulation, the system is required to organize the label space so that the most plausible categories appear at the top of the ranking.

This view is more appropriate than a pure top-1 classification approach because queries are often short, ambiguous, or only partially informative. In some cases, the available text does not contain enough information to support a unique code assignment. For example, a query may refer to the sale of a product without making clear whether the activity concerns wholesale or retail trade, even though this distinction is essential for correct coding. More generally, the role of the system is not only to identify plausible categories, but also to support the clarification of what additional information may be needed for a reliable coding decision.

## 3.2 Coding as a ranking problem

Framing the task as ranking makes the role of retrieval explicit. The system receives a query and compares it with the textual representations associated with the categories of the classification. The output is therefore an ordered list of candidates rather than an immediate final decision. In this setting, the quality of the coding system depends on whether the correct or most appropriate category appears among the top-ranked candidates, and not only on whether it is placed in the first position.

For this reason, evaluation measures based on top-k retrieval are more closely aligned with the intended use of the system than exact-match accuracy alone. The ranking formulation also provides a natural setting for considering how coding outputs may later be filtered, interpreted, or used with caution under operational constraints.

Under this formulation, the contribution of the paper is not to present a fully autonomous coding system that replaces statistical judgment. Rather, it is to assess whether embedding-based semantic retrieval can provide a defensible and operationally relevant framework for supporting economic-activity coding in Official Statistics. The next section turns this formulation into a concrete methodological design by describing how the classification is represented, how similarity is computed, and how performance is evaluated.
