# Methodology

## Problem formulation

The objective of this study is the automatic assignment of CP2021 occupational codes to Italian free-text job descriptions. Rather than formulating the task as a standard multiclass classification problem, we cast it as a semantic retrieval problem. This choice is motivated by the structure of the task itself: both the input and the label space are textual, and the quality of the assignment depends on the semantic correspondence between a user-generated occupation description and a textual representation of the target occupational category.

Let \( q \in \mathcal{Q} \) denote a query describing an occupation, and let \( D = \{d_1, d_2, \dots, d_N\} \) be the set of candidate class descriptors. Each descriptor \( d_j \) is a natural-language representation of one CP2021 occupational class. A sentence-embedding model defines a mapping

\[
f : \mathcal{T} \rightarrow \mathbb{R}^m,
\]

where \( \mathcal{T} \) is the space of input texts and \( \mathbb{R}^m \) is a dense semantic vector space. Query texts and class descriptors are projected into the same space:

\[
\mathbf{q} = f(q), \qquad \mathbf{d}_j = f(d_j).
\]

Relevance is estimated through a similarity function \( s(\mathbf{q}, \mathbf{d}_j) \), implemented here through cosine similarity:

\[
s(\mathbf{q}, \mathbf{d}_j) = \frac{\mathbf{q} \cdot \mathbf{d}_j}{\lVert \mathbf{q} \rVert \lVert \mathbf{d}_j \rVert }.
\]

For each query, the system ranks all candidate descriptors in decreasing order of similarity and returns the top-\(k\) results:

\[
\pi_k(q) = \operatorname{TopK}_{d_j \in D} \; s(\mathbf{q}, \mathbf{d}_j).
\]

Under this formulation, occupational coding becomes a ranking problem over a semantically structured label space. The correct code is not predicted through a dedicated classifier head; instead, it is recovered if the corresponding descriptor is placed sufficiently high in the ranking.

This formulation is especially suitable for occupational coding for two reasons. First, it preserves the textual nature of the classification system and makes the representation of classes analytically explicit. Second, it naturally supports top-\(k\) evaluation and human-in-the-loop coding workflows, where a small set of highly ranked candidates is often more useful than a single forced prediction.

## Sentence embeddings and semantic search

The central methodological assumption of the study is that semantically similar occupation descriptions and occupational categories should be mapped to nearby points in the embedding space. In this framework, the effectiveness of the coding system depends on two components:

1. the quality of the embedding function \(f\), that is, the model’s ability to produce semantically informative sentence representations
2. the quality of the descriptor text \(d_j\), that is, the extent to which each occupational class is verbalized in a way that is semantically compatible with real user queries

This second point is central to the present work. In institutional classification systems, official category descriptions are often concise, formal, and lexically distant from the language used in survey responses or user-generated text. As a consequence, even strong multilingual embedding models may fail to align queries and classes if the descriptor representation is too narrow or too formalized.

The study therefore treats descriptor construction as a substantive methodological variable rather than a purely technical preprocessing step. The main hypothesis is that retrieval quality can be improved not only by selecting a stronger embedding model, but also by designing richer descriptors that reduce the semantic mismatch between user language and classification language.

## Data setup

The benchmark is evaluated at the 5-digit CP2021 level. The query side is defined by a labelled dataset of 61,881 Italian occupation descriptions paired with gold-standard CP2021 codes, while the descriptor side is constructed from the official classification and an auxiliary resource containing 6-digit occupational entries linked to their parent 5-digit categories. These resources make it possible to construct alternative descriptor formulations with increasing amounts of semantic detail.

## Descriptor design

The study evaluates three descriptor configurations, all defined at the 5-digit CP2021 level.

### Official descriptors

The first configuration, denoted `official`, uses only the official 5-digit class representation. Each descriptor combines the official occupational title and the official class description. This condition acts as the conceptual baseline because it uses only the wording provided by the classification scheme itself.

### Descriptors enriched with 6-digit occupational examples

The second configuration, denoted `enriched`, augments each 5-digit descriptor with the occupational examples derived from the associated 6-digit entries. The rationale is that 6-digit labels provide a richer and more concrete lexical surface than the official 5-digit description alone. While the 5-digit description captures the conceptual definition of the class, the 6-digit entries expose the occupational vocabulary that typically instantiates that class in practice.

From the perspective of semantic retrieval, this enrichment increases the probability that at least part of the lexical and semantic content of a user query will overlap with the class descriptor in the embedding space.

### Descriptors enriched with query-derived lexical evidence

The third configuration, denoted `tf-idf`, adds a further layer of enrichment by incorporating representative terms extracted from the labelled query corpus. This enrichment is derived from a TF-IDF-based analysis conducted at the 5-digit code level. For each class, the procedure identifies the terms that are most characteristic of the queries assigned to that class and appends them to the descriptor text.

More formally, let \( c \) denote a 5-digit occupational class and let \( t \) denote a candidate term extracted from the set of queries assigned to that class. The TF-IDF score used for term selection follows the standard intuition:

\[
\operatorname{tfidf}(t,c) = \operatorname{tf}(t,c)\cdot \operatorname{idf}(t),
\]

where \(\operatorname{tf}(t,c)\) captures the relative importance of term \(t\) within the query set of class \(c\), and \(\operatorname{idf}(t)\) downweights terms that are broadly shared across many classes. In operational terms, the method retains the highest-scoring class-specific terms for each class and concatenates them to the corresponding descriptor text.

The conceptual role of this step is to inject into the class representation the lexical evidence that emerges from actual user descriptions. In other words, the official classification is supplemented with a weakly supervised lexical view of how each occupation is expressed in the empirical data. This is intended to reduce the gap between institutional terminology and observed occupational language and is precisely what distinguishes the `tf-idf` descriptor from the simpler `enriched` configuration.

## Query-side representation

Each record in the labelled dataset contributes one query text and one target CP2021 code. Query texts are normalized through a minimal preprocessing pipeline based on trimming and lowercasing. The target code is then evaluated at the granularity required by the descriptor space, which in the main benchmark corresponds to the 5-digit level.

This deliberately light preprocessing reflects the intended use case of the system: the retrieval model should operate on realistic occupation descriptions with minimal manual normalization. The objective is not to optimize a handcrafted text-cleaning pipeline, but to evaluate how far dense semantic representations and descriptor design can go under relatively natural input conditions.

## Benchmark design

The evaluation compares multiple multilingual sentence-embedding models under the same retrieval protocol. The benchmark includes:

1. `BAAI/bge-m3`
2. `intfloat/multilingual-e5-base`
3. `intfloat/multilingual-e5-large`
4. `Alibaba-NLP/gte-multilingual-base`
5. `Qwen/Qwen3-Embedding-0.6B`
6. `Qwen/Qwen3-Embedding-4B`
7. `Qwen/Qwen3-Embedding-8B`

For each combination of embedding model and descriptor configuration, the full set of class descriptors is embedded once to create the retrieval index, and each query is then encoded and matched against the indexed descriptors. Retrieval is truncated at the top 100 candidates.

In the case of the E5 family, the query and document prefixes recommended by the model family are applied to preserve the intended query-document asymmetry. The other models are evaluated on the normalized text directly.

## Ranking-based evaluation

Let \( y_i \) be the gold-standard code associated with query \(q_i\), and let \( c(d_{ir}) \) be the code associated with the descriptor retrieved at rank \(r\). The rank of the correct code is defined as

\[
\operatorname{rank}(q_i) = \min \{ r \in \{1,\dots,K\} : c(d_{ir}) = y_i \},
\]

with rank set to 0 when the correct code does not appear in the retrieved list.

The evaluation uses standard ranking metrics. Accuracy@\(k\) is defined as

\[
\operatorname{Accuracy@k} = \frac{1}{n} \sum_{i=1}^{n} \mathbf{1}\{\operatorname{rank}(q_i) \in \{1,\dots,k\}\},
\]

where \(n\) is the number of queries and \(\mathbf{1}\{\cdot\}\) is the indicator function. Mean Reciprocal Rank is defined as

\[
\operatorname{MRR} = \frac{1}{n} \sum_{i=1}^{n}
\begin{cases}
\frac{1}{\operatorname{rank}(q_i)} & \text{if } \operatorname{rank}(q_i) > 0, \\
0 & \text{otherwise.}
\end{cases}
\]

In addition, the analysis reports mean rank, median rank, and not found rate, that is, the proportion of queries for which the gold-standard code does not appear in the top-100 retrieved candidates.

These metrics capture complementary aspects of system quality. Accuracy@1 measures exact retrieval at the first position; Accuracy@\(k\) for small \(k\) measures the practical usefulness of a short candidate list; MRR captures the overall position of the correct answer in the ranking; and not found rate quantifies complete retrieval failures.

## Analytical strategy

The empirical analysis is designed to answer two questions. The first is model-oriented: which multilingual embedding model produces the strongest semantic alignment between occupation descriptions and CP2021 classes? The second is representation-oriented: how much does retrieval quality change when occupational descriptors are progressively enriched?

The first question is addressed by comparing models within each descriptor setting. The second is addressed by holding the model fixed and comparing the three descriptor formulations, from the official baseline to the progressively enriched variants.

This design makes it possible to isolate a key contribution of the study. The paper does not treat class descriptors as fixed inputs, but as an explicit object of methodological design. The benchmark therefore evaluates not only model performance, but also the extent to which better verbalizations of occupational classes improve semantic retrieval.

## Scope of the method

The present methodology should be understood as a retrieval benchmark rather than a complete production evaluation. Its purpose is to measure how embedding-based semantic search behaves under controlled variations in descriptor design and model choice. It does not yet include significance testing, subgroup-level error analysis, or qualitative inspection of failure cases. These elements would be valuable extensions, but the current study is primarily concerned with establishing whether descriptor enrichment produces a clear and robust gain in retrieval-based occupational coding.
